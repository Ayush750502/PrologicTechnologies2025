# IOS Requirements for publishing application:

Before publishing a new iOS app to the App Store, ensure you have:

* **Apple Developer Program membership** (active paid account).
* **App Store Connect** app created with a unique Bundle ID.
* **Release-ready app**:

  * No crashes or major bugs.
  * Production API endpoints.
  * Tested on real devices.
  * Valid version and build numbers.
* **Required assets**:

  * 1024 × 1024 App Icon.
  * Required screenshots for supported devices.
* **App Store metadata**:

  * App name.
  * Description.
  * Keywords.
  * Category.
  * Age rating.
  * Copyright.
* **Required URLs**:

  * Privacy Policy URL.
  * Support URL.
  * Marketing URL (optional).
* **Privacy & permissions**:

  * Complete App Privacy questionnaire.
  * Include `PrivacyInfo.xcprivacy` if required.
  * Add clear usage descriptions for all requested permissions (Camera, Photos, Microphone, etc.).
* **Review information**:

  * Demo/test account (if login is required).
  * Review notes explaining how to access app features.
* **Build submission**:

  * Archive and upload the app from Xcode.
  * Select the uploaded build in App Store Connect.
  * Submit for Apple Review.

**Final checklist:** Developer Account ✓ | Release Build ✓ | App Icon ✓ | Screenshots ✓ | Metadata ✓ | Privacy Policy ✓ | App Privacy ✓ | Permissions ✓ | Demo Account (if needed) ✓ | Review Notes ✓ | Submit for Review ✓
